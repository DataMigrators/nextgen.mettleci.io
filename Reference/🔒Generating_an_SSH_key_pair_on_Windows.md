# 🔒Generating an SSH key pair on Windows

# TL;DR

Usually the process preached is to generate SSH keys using PuTTYgen,
copy the public key to Bitbucket and you’ll be able to connect. But is
not the case. The keys generated by PuTTYgen are not readable by
BitBucket. In this article i’ll be discussing the alternative approach —
Using GitBash

# GitBash

Git Bash comes included as part of
<a href="https://gitforwindows.org/" rel="nofollow"><u>Git for
Windows</u></a>.
<a href="https://www.atlassian.com/git/tutorials/git-bash"
rel="nofollow"><u>More here</u></a>

1.  Load GitBash

2.  Type the command `ssh-keygen`

3.  This will start generating the keys. Press `enter` to use defaults.

`eval $(ssh-agent)` and `ssh-add ~/.ssh/<private-key>`

1.  Open the public key and copy the text.

`cat id_rsa.pub`

Login to your account → Click on profile icon on top-right → Manage
Account → SSK Keys → Add key

# Test Access

Open a new folder and try this command

`$ git clone ssh://<url from bitbucket repository>`

Since the key is added to your system and to Bitbucket, you should be
able to clone and access the repository.
